# ADR-003: Offline-first processing with a labeled near-real-time demo mode

- **Status:** Accepted
- **Date:** 2026-07-07
- **Decision owner:** TrafficPulse project lead (maintainer role)
- **Supersedes:** none
- **References:** `docs/architecture-review.md` §10, §22, §23; `docs/evaluation-protocol.md` §19–§20, §23

## Context

On the target hardware (RTX 4060 Laptop, 8 GB VRAM, 16 GB RAM), the full
concurrent stack (detector + helmet + occupancy + signal + ANPR) is **not**
expected to run in real time (architecture §10). The evaluated system must
therefore be defined honestly, and any "real-time" language must be scoped and
labeled (spec-revision note #1; architecture §22).

This ADR records a **decision only**; it implements no pipeline.

## Decision drivers

- Compute reality of the target hardware (architecture §10 — none of it a
  measured claim yet).
- Honesty: no real-time claim for the full stack before measurement.
- A demonstrable interactive mode for the capstone.

## Considered options

- **Offline-first + labeled near-real-time demo (chosen).**
- Claiming a real-time integrated pipeline — rejected: unsupported on the target
  hardware and dishonest before measurement.
- Offline-only with no interactive demo — rejected: weakens the demonstration.

## Decision

- **Primary and evaluated mode is offline batch** (video file in, events out).
- A **near-real-time demo mode** runs only **detector + tracker + rules on a
  single stream**, with ANPR and evidence rendering asynchronous.
- "Real-time" language is **confined to the demo mode and always labeled**; it is
  never claimed for the full concurrent stack.

## Consequences

- The U4 evaluation protocol measures **offline throughput** and the **demo mode
  separately** (evaluation-protocol §19–§20), and reports numbers only **after
  measurement**.
- Night remains a **robustness-analysis slice only**, not a supported operating
  condition (evaluation-protocol §23) — consistent with this offline-first,
  honesty-first posture.

## Risks and mitigations

- **Demo-mode performance is still unproven** — it is a Phase 1 measurement, not
  a Phase 0-F claim. *Mitigation:* label it, measure it, report it honestly.

## Revisit triggers

- Hardware upgrade or measured performance that enables more concurrency.
- A future requirement for streaming/near-real-time evaluation.
